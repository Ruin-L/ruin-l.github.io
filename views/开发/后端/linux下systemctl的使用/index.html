Linux 下 systemctl 的使用 ​

systemctl 是 Linux 系统中用于管理系统服务的命令行工具，它是 systemd 初始化系统的主要命令行工具。systemd 是许多现代 Linux 发行版（如 CentOS 7+、Ubuntu 16.04+、Debian 8+ 等）的默认初始化系统，用于启动、停止和管理系统服务。

systemctl 的基本概念 ​

单元（Unit） ​

systemd 将需要管理的对象称为"单元"（Unit），主要包括：

  • service：系统服务
  • socket：进程间通信的套接字
  • device：硬件设备
  • mount：文件系统挂载点
  • automount：自动挂载点
  • target：一组单元的集合（类似于运行级别）
  • timer：定时器

服务状态 ​

服务可以处于以下几种状态：

  • active (running)：服务正在运行
  • active (exited)：服务已成功完成一次性配置
  • active (waiting)：服务正在运行但在等待事件
  • inactive (dead)：服务没有运行
  • failed：服务启动失败

systemctl 基本命令 ​

查看服务状态 ​

bash
# 查看服务状态
systemctl status nginx.service

# 检查服务是否正在运行
systemctl is-active nginx.service

# 检查服务是否开机自启
systemctl is-enabled nginx.service

# 列出所有运行中的服务
systemctl list-units --type=service --state=running

管理服务 ​

bash
# 启动服务
systemctl start nginx.service

# 停止服务
systemctl stop nginx.service

# 重启服务
systemctl restart nginx.service

# 重新加载配置文件（不中断服务）
systemctl reload nginx.service

# 设置开机自启
systemctl enable nginx.service

# 禁止开机自启
systemctl disable nginx.service

# 重新加载 systemd 配置
systemctl daemon-reload

创建自定义服务 ​

服务单元文件结构 ​

服务单元文件通常位于 /etc/systemd/system/ 或 /usr/lib/systemd/system/ 目录下，扩展名为 .service。

一个典型的服务单元文件包含三个部分：

  1. [Unit]：描述服务的基本信息、依赖关系和启动顺序
  2. [Service]：定义服务的具体行为
  3. [Install]：定义服务的安装信息

示例：创建一个简单的 Web 应用服务 ​

  1. 创建服务单元文件：
bash
sudo nano /etc/systemd/system/myapp.service
1
  1. 添加以下内容：
ini
[Unit]
Description=My Web Application
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/var/www/myapp
ExecStart=/usr/bin/node /var/www/myapp/app.js
Restart=on-failure
RestartSec=5s
StandardOutput=syslog
StandardError=syslog
SyslogIdentifier=myapp

[Install]
WantedBy=multi-user.target

常见服务管理场景 ​

查看服务日志 ​

bash
# 查看指定服务的日志
journalctl -u nginx.service

# 查看最近的日志
journalctl -u nginx.service -n 50

# 实时查看日志
journalctl -u nginx.service -f

服务资源限制 ​

在服务单元文件的 [Service] 部分可以添加资源限制：

ini
[Service]
# 限制 CPU 使用率
CPUQuota=50%

# 限制内存使用
MemoryLimit=1G

# 限制进程数
LimitNPROC=10

常见问题排查 ​

  1. 服务启动失败：

    bash
    systemctl status myapp.service
journalctl -u myapp.service -n 50
    1
    2

  2. 服务依赖问题：

    bash
    systemctl list-dependencies --all myapp.service
    1

  3. 权限问题： 检查服务单元文件中的 User 和 Group 设置，以及工作目录和执行文件的权限。

总结 ​

systemctl 是现代 Linux 系统中管理服务的强大工具，掌握它的使用可以帮助我们更高效地管理系统服务。通过本文介绍的基本命令和示例，你应该能够处理大多数服务管理任务。

对于更复杂的场景，可以参考 systemd 的官方文档或使用 man systemd.service 命令查看详细说明。
