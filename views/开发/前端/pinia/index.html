Pinia 介绍 ​

Pinia 是 Vue.js 的官方状态管理库（Vue 3+ 推荐），由 Vue 核心团队维护。它解决了 Vuex 中的一些痛点，提供更简洁、直观且类型安全的 API，主要特点包括：

  1. 轻量级 ：体积小（约 1KB），无冗余设计。
  2. DevTools 支持 ：与 Vue DevTools 深度集成。
  3. Composition API 友好 ：天然支持 Vue 3 的 Composition API。
  4. TypeScript 优先 ：提供完整的类型推断。
  5. 模块化设计 ：无需嵌套模块，每个 Store 独立存在。
  6. 无 mutations ：直接修改状态或使用 actions 处理异步。

3. 定义 Store ​

javascript

js
// stores/counter.js
import { defineStore } from 'pinia'

export const useCounterStore = defineStore('counter', {
  // 状态（相当于 data）
  state: () => ({
    count: 0,
    name: 'Pinia User'
  }),

  // 计算属性（相当于 computed）
  getters: {
    doubleCount: (state) => state.count * 2,
    // 访问其他 getter
    greeting: (state) => `Hello, ${state.name}! Count is ${this.doubleCount}`
  },

  // 方法（支持同步/异步）
  actions: {
    increment() {
      this.count++ // 直接通过 this 修改状态
    },
    async fetchData() {
      const res = await api.getData()
      this.name = res.data.name
    }
  }
})

4. 在组件中使用 ​

vue
<script setup>
import { useCounterStore } from '@/stores/counter'

const counter = useCounterStore()

// 访问状态
console.log(counter.count)

// 使用 getter
console.log(counter.doubleCount)

// 调用 action
counter.increment()

// 直接修改状态（需谨慎）
counter.count = 5

// 批量修改（推荐）
counter.$patch({
  count: 10,
  name: 'Updated Name'
})

// 重置状态
counter.$reset()
</script>

<template>
  <p>{{ counter.count }}</p>
  <p>{{ counter.doubleCount }}</p>
  <button @click="counter.increment">+1</button>
</template>

持久化工具：pinia-plugin-persistedstate ​

Pinia 本身不包含持久化功能，需通过插件实现。推荐使用官方维护的 pinia-plugin-persistedstate 。

安装 ​

bash

bash
npm install pinia-plugin-persistedstate
# 或
yarn add pinia-plugin-persistedstate
1
2
3

配置 ​

javascript

js
// main.js
import { createApp } from 'vue'
import { createPinia } from 'pinia'
import piniaPluginPersistedstate from 'pinia-plugin-persistedstate'

const pinia = createPinia()
pinia.use(piniaPluginPersistedstate) // 注入插件

createApp(App).use(pinia).mount('#app')

使用持久化 ​

在 Store 中添加 persist 配置：

javascript

js
export const useCounterStore = defineStore('counter', {
  state: () => ({ count: 0 }),

  // 开启持久化（默认 localStorage）
  persist: true

  // 高级配置
  // persist: {
  //   key: 'my-custom-key',         // 存储键名
  //   storage: sessionStorage,      // 更换为 sessionStorage
  //   paths: ['count'],             // 只持久化 count 字段
  //   serializer: {                 // 自定义序列化
  //     serialize: JSON.stringify,
  //     deserialize: JSON.parse
  //   }
  // }
})

特性说明 ​

配置项说明
key存储的键名（默认：store id）
storage存储引擎（默认：localStorage，可换为 sessionStorage 等）
paths指定要持久化的状态字段（数组）
beforeRestore恢复状态前的钩子函数（ctx: PiniaPluginContext）
afterRestore恢复状态后的钩子函数
serializer自定义序列化方法（默认使用 JSON.stringify/parse）

最佳实践 ​

  1. 命名规范 ：Store 命名使用 useXxxStore（如 useUserStore）
  2. 组合 Stores ：可在 Store 中导入其他 Store javascript
js
// stores/user.js
import { useCartStore } from './cart'

export const useUserStore = defineStore('user', {
  actions: {
    checkout() {
      const cart = useCartStore()
      cart.clearCart()
    }
  }
})

  1. SSR 注意 ：持久化插件在服务端渲染（SSR）中需特殊处理
