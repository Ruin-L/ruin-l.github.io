Axios 是什么？ ​

Axios 是一个非常流行、强大且 基于 Promise 的 HTTP 客户端库 ，主要用于在浏览器和 Node.js 环境中发起 HTTP 请求（如 GET, POST, PUT, DELETE 等）。它的设计目标是让发送异步 HTTP 请求和处理响应变得简单、直观和一致。

为什么 Axios 如此受欢迎？ ​

相较于原生的 fetch API 或早期的 XMLHttpRequest，Axios 提供了许多开箱即用的便利功能和更友好的 API，解决了开发中的常见痛点：

  1. 基于 Promise：
    • 使用 ES6 Promises 处理异步操作，避免了回调地狱，可以使用 async/await 语法糖，让异步代码更清晰、易读、易维护。
  2. 浏览器 & Node.js 双环境支持：
    • 一份代码，既能在浏览器中运行，也能在 Node.js 服务器端运行，提供了统一的 API。
  3. 请求/响应拦截器：
    • 请求拦截器： 在请求发送前进行操作（例如：添加认证 Token Authorization 头、设置公共参数、修改请求配置、记录日志）。
    • 响应拦截器： 在响应到达 then/catch 之前进行操作（例如：统一处理错误状态码、转换响应数据格式、提取核心数据）。
    • 拦截器极大地提高了代码复用性和可维护性。
  4. 自动转换 JSON 数据：
    • 自动将发送的 JavaScript 对象序列化为 JSON（设置 Content-Type: application/json）。
    • 自动将接收到的 JSON 响应数据解析为 JavaScript 对象。省去了手动 JSON.stringify() 和 JSON.parse() 的步骤。
  5. 客户端防御 XSRF (CSRF)：
    • 内置支持防御跨站请求伪造攻击。可以配置在请求中自动携带特定的 Cookie 或 Header（通常名为 X-XSRF-TOKEN），与后端配合实现防护。
  6. 取消请求：
    • 提供了清晰易用的机制来取消正在进行的 HTTP 请求（基于 CancelToken 或较新的 AbortController ）。这在用户离开页面、组件卸载或需要中断长时间请求时非常有用。
  7. 进度监控：
    • 支持监听上传和下载的进度事件，方便实现进度条等功能。
  8. 更强大的错误处理：
    • 能区分网络错误（如无网络连接）和 HTTP 状态码错误（如 404, 500）。错误响应对象包含了丰富的错误信息（状态码、状态文本、请求配置、响应数据等），便于定位问题。
  9. 并发请求：
    • 提供 axios.all() 和 axios.spread() 方法（现在更推荐使用 Promise.all()）来方便地处理多个并发请求。
  10. 简洁一致的 API：
    • 方法命名清晰：axios.get(), axios.post(), axios.put(), axios.delete(), axios.request(config) 等。
    • 配置对象（config）统一管理所有请求参数（URL, method, headers, data, params, timeout 等）。
  11. TypeScript 支持：
    • 提供了优秀的 TypeScript 类型定义，增强了开发体验和代码安全性。

  7. 拦截器示例 (添加 Token)：javascript

    javascript
    // 请求拦截器
axios.interceptors.request.use(
  (config) => {
    // 从安全存储（如 localStorage, Vuex, Redux, cookie）获取 token
    const token = localStorage.getItem('userToken')
    if (token) {
      config.headers.Authorization = `Bearer ${token}`
    }
    return config // 必须返回修改后的 config
  },
  (error) => {
    return Promise.reject(error) // 对请求错误做些什么
  }
)

// 响应拦截器 (示例：处理通用错误)
axios.interceptors.response.use(
  (response) => {
    return response // 直接返回成功响应
  },
  (error) => {
    if (error.response) {
      // 服务器有响应，但状态码不在 2xx 范围内
      switch (error.response.status) {
        case 401:
          // 未授权，跳转到登录页
          router.push('/login')
          break
        case 403:
          // 禁止访问
          alert('权限不足')
          break
        case 500:
          // 服务器内部错误
          alert('服务器开小差了，请稍后再试')
          break
        // ... 其他状态码处理
      }
    } else if (error.request) {
      // 请求已发出，但没有收到响应 (网络错误)
      console.error('网络错误:', error.request)
    } else {
      // 设置请求时发生错误
      console.error('请求配置错误:', error.message)
    }
    return Promise.reject(error) // 将错误传递下去，以便在具体请求的 catch 中还能处理
  }
)

通用请求类 ​

typescript
import axios, { AxiosInstance, AxiosRequestConfig, AxiosResponse } from 'axios'
import { ElMessage, ElMessageBox } from 'element-plus'
// 创建一个新的axios实例
const instance: AxiosInstance = axios.create({
  baseURL: import.meta.env.VITE_BASE_API,
  timeout: 60000,
  withCredentials: false,
  headers: {
    'Content-Type': 'application/json'
  }
})

// 请求拦截器
instance.interceptors.request.use(
  (config: any) => {
    // 在发送请求之前做些什么
    // 例如，添加token到请求头
    // const token = localStorage.getItem('token');
    // if (token) {
    //   config.headers.Authorization = `Bearer ${token}`;
    // }
    return config
  },
  (error: any) => {
    // 对请求错误做些什么
    return Promise.reject(error)
  }
)

// 响应拦截器
instance.interceptors.response.use(
  (response: AxiosResponse) => {
    // 对响应数据做点什么
    console.log(process.env.NODE_ENV, '变量')

    const data = response.data

    // 判断接口返回的 Message 字段是否为 Success
    if (data && data.Message !== 'Success') {
      // 如果不是 Success，则将请求视为失败
      ElMessageBox.alert(JSON.stringify(data), '错误', {
        confirmButtonText: '确定',
        type: 'error',
        dangerouslyUseHTMLString: false
      })
      return Promise.reject({
        response: {
          status: 200,
          data: data,
          message: JSON.stringify(data) || '请求失败'
        }
      })
    }

    // 如果是 Success，则返回数据
    return data
  },
  (error: any) => {
    // 对响应错误做点什么
    if (error.response) {
      // 请求已发出，但服务器响应的状态码不在 2xx 范围内

      console.error('请求错误:', error.response.status, error.response.data)
    } else if (error.request) {
      // 请求已经发出，但没有收到响应

      console.error('网络错误:', error.request)
    } else {
      // 在设置请求时发生了一些事情，触发了一个错误
      console.error('错误:', error.message)
    }
    ElMessageBox.alert(JSON.stringify(error.request), '请求错误', {
      confirmButtonText: '确定',
      type: 'error',
      dangerouslyUseHTMLString: false
    })
    return Promise.reject(error)
  }
)

export default instance

总结 ​

Axios 是现代 Web 开发中进行 HTTP 通信的 首选工具之一 。它通过提供基于 Promise 的 API、拦截器、自动 JSON 转换、取消请求、XSRF 防护、统一的浏览器/Node.js 支持等强大功能，极大地简化了 HTTP 请求的发送和响应的处理过程，提高了开发效率和代码质量。其清晰一致的 API 设计也降低了学习成本。如果你需要在项目中处理网络请求，Axios 是一个非常值得考虑和采用的库。
